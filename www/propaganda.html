Subversion Propaganda

Here are Subversion testimonials, and a list of prominent projects using Subversion. If you're trying to persuade your organization to try Subversion, this page might help you.

Open Source Projects using Subversion

    Project site: http://www.apache.org
    Repository: http://svn.apache.org/repos/asf/
    "The Apache Software Foundation provides support for the Apache community of open-source software projects."

  • Samba: SMB services for *nix systems.

    Project site: http://www.samba.org
    Repository: http://websvn.samba.org/
    "Samba is an Open Source/Free Software suite that provides seamless file and print services to SMB/CIFS clients."

  • Zope: web application server/framework.

    Project site: http://www.zope.org
    Repository: http://svn.zope.org/
    "Zope is an open source application server for building content managements, intranets, portals, and custom applications."

  • Xiph: open-source multimedia protocols.

    Project site: http://www.xiph.org
    Repository: http://svn.xiph.org/
    "Xiph.Org Foundation is a non-profit corporation dedicated to protecting the foundations of Internet multimedia from control by private interests. Our purpose is to support and develop free, open protocols and software to serve the public, developer and business markets."

  • Debian: a popular Linux distribution.

    Project site: http://www.debian.org
    Repository: http://svn.debian.org/
    "Debian is a free operating system (OS) for your computer. [...] Debian uses the Linux kernel (the core of an operating system), but most of the basic OS tools come from the GNU project; hence the name GNU/Linux."

  • Conectiva: a South American Linux distribution.

    Project site: http://www.conectiva.com.br
    Repository: https://moin.conectiva.com.br/RepositorySystem
    "A Conectiva, maior empresa em serviços Linux na América Latina, oferece uma linha de produtos e serviços que atende com eficácia os usuários domésticos, pequenas e médias empresas, grandes corporações e órgãos governamentais."

  • Trac: a project management system.

    Project site: http://projects.edgewall.com/trac/
    Repository: http://svn.edgewall.com/repos/trac/
    "Trac is a minimalistic web-based software project management and bug/issue tracking system. It provides an interface to revision control systems (Subversion), an integrated Wiki and convenient report facilities.

Testimonials

  • Post from Stuart Robertson of Absolute Systems, Inc

    I introduced SVN to Absolute Systems Inc. (www.absolutesys.com) where I work about a year ago, and for about 8 months we ran internal SVN pilots, played around to gain experience and trust, etc.

    In the last 4 months we have migrated all of our internal product source repositories from Visual Source-Safe to SVN using an internally-written VSS-to-SVN migration tool.

    Our largest SVN repository is now 3.7GB and currently has 68806 revisions. We are running SVN 1.0.1 + Apache 2.0.48 on Linux. ...

    SVN is a superb piece of work, and it is a *huge* step forward from VSS. To put things in perspective... previously we had 26 VSS databases for one product, primarily because of problems with VSS when the repositories grow large. As you can imagine, trying to manage product releases across so many repositories was really painful.

    Now, with SVN, *all* of the artifacts for that same product are in a single repository, meaning that with a few cheap copy operations all of the sources that make up a given release can be grouped together. ...

  • Post from Sebastien Cevey

    ... I've been using Subversion for personnal and team projects for a few years now. I never regretted it, except for the deep frustration I feel everytime I have to go back to CVS.

    I am glad to have introduced many people to this software, through projects, talks or a course on versionning systems. I haven't heard anybody disappointed by it, and this doesn't surprise me. ...

  • From "culley", OS news article
    I recently evaluated both subversion and arch and chose subversion for my company. Probably the number 1 reason was the subversion book-- arch is not documented as well as subversion. All I want a centralized repository where me and 2-3 other developers can collaborate. I don't want to push the envelope on version control-- the existing models work great for me and 99% of the open source projects out there. I don't need to create branches while on an air plane. I do need to easily move files around in my repository. Subversion it is!
