Last Week's Activities

Milestone 2 contains two big leaps: a working repository filesystem and a working network layer. Over the last two months, the Chicago team (Karl, Mike, Ben) completed the filesystem (minus certain optimizations) and it can be hooked directly to the client for "local" checkouts, commits, and updates. This week, Greg Stein is finishing the DAV layer. When we're able to do checkouts, commits, and updates through Apache, we'll tag the code as 'M2' and move on.

Meanwhile, our many meetings in SF have resulted in a comprehensive "to-do" list to reach a feature-complete alpha. (See notes/alpha_checklist in our source tree.)

Last week, we worked on squashing existing memory bugs and implementing a 'black-box' testing system in python. This week, we'll be re-architecting some internals which will allow client subcommands to accept multiple target paths.
