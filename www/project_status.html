Last Week's Activities

Checkouts and commits are working now. The client can connect to the repository layer (via the ra_local library, not the network), check out a tree, and commit changes, including: file modifications, file adds, file deletes, dir adds, and dir deletes. Copies work on the repository side, but still need further tweaks in the client.

Next week, we'll parallelize: Greg Stein will make the above happen over the network, while the client/filesystem team works on updates. Updates are a matter of getting the client to report its "working copy state" to the server, and having the server send back an appropriate delta to bring that working copy up-to-date.
