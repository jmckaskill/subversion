Inversion Project Mailing Lists

To subscribe or unsubscribe, send an email message to the email address associated with each "subscribe" or "unsubscribe" line below. The subject line and message body are not used, so you may leave them blank.

You must subscribe before you can post to any of these lists. This rule helps to prevent spam.

announce@subversion: Announcements of new releases and other major project events.
Archive: view on-line
Subscribe: announce-subscribe@subversion.tigris.org
Unsubscribe: announce-unsubscribe@subversion.tigris.org
 
dev@subversion: The main list for developers to discuss development issues, and general discussion of requirements for version control in open source projects.
Archive: view on-line
Subscribe: dev-subscribe@subversion.tigris.org
Unsubscribe: dev-unsubscribe@subversion.tigris.org
 
bugs@subversion: Automated notifications of bugs reported against subversion and discussion of those bugs.
Archive: view on-line
Subscribe: bugs-subscribe@subversion.tigris.org
Unsubscribe: bugs-unsubscribe@subversion.tigris.org
 
cvs@subversion: Automated notifications of source code changes and discussion of those changes.
Archive: view on-line
Subscribe: cvs-subscribe@subversion.tigris.org
Unsubscribe: cvs-unsubscribe@subversion.tigris.org
