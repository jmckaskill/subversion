The Future

This list is a rough prioritization of planned feature additions, based on an ever-changing consensus of the Subversion developer community. Because this is a volunteer open-source project, it's hard to predict exact dates or timetables for these new features. At most, we can express dependencies and predict the order in which things will be worked on. The best way to track development is to subscribe to the developer's mailing list.

  • Subversion 1.3, 1.4, and 1.5 Goals:
    • TBD (under discussion, watch this space for updates)
  • Medium-term Goals:
    • true rename support (not based on copy/delete)
    • merge tracking (describes a whole class of problems)
    • repository-level ACLs
  • Other Long-term Goals
    • SQL repository back-end
    • rewrite of working-copy library
    • broader WebDAV/deltaV compatibility
    • pluggable client-side diff programs
    • progressive multi-lingual support
